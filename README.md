## Дисклеймер

Данный репозиторий содержит результаты **исследования безопасности поведения API** в продакшен-среде mimolet.

Не содержит:
- автоматизированных инструментов эксплуатации
- извлечения персональных данных
- атакующих скриптов

Цель — ответственное раскрытие проблем бизнес-логики и антиабьюз механизмов.

## Краткое описание проблем

### 1. Отсутствие серверных ограничений частоты действий

Взаимодействия пользователя с системой не ограничиваются на серверной стороне.

- отсутствует контроль частоты действий
- отсутствует серверная валидация
- нет механизма anti-replay

---

### 2. Отсутствие одноразовости action-токенов

Механизм action token не реализует строгую модель одноразового использования.

- токены остаются валидными после выполнения действия
- повторное выполнение действия с тем же токеном успешно обрабатывается

---

### 3. Слабая серверная валидация бизнес-логики

Серверная часть не полностью контролирует состояние взаимодействий:

- отсутствует строгая идемпотентность действий
- клиентская логика частично определяет ограничения
- сервер принимает валидные повторяющиеся операции без блокировки

---

### 4. Логика выдачи ленты

API выдачи карточек работает без строгой привязки к состоянию потребления:

- возможна манипуляция параметрами выдачи
- отсутствует жёсткая синхронизация состояния просмотра
- контроль очереди частично реализован на стороне клиента

---

## Влияние

Выявленные особенности могут приводить к:

- обходу ограничений взаимодействий
- автоматизации действий пользователя
- искажению метрик активности
- деградации качества рекомендаций
- нарушению предполагаемой бизнес-логики платформы

---

## Оценка критичности

- Тип: Business Logic Abuse
- Уровень: High
- CVSS: 7.5

---

## Рекомендации для разработчиков

- внедрить серверный rate limiting для действий пользователя
- реализовать строгую идемпотентность interaction endpoints
- сделать action token одноразовым (state transition модель)
- перенести ограничения полностью на серверную сторону
- синхронизировать бизнес-лимиты с backend логикой
